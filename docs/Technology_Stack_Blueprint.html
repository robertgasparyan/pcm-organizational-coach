1Project Overview

A full-stack PCM (Process Communication Model) organizational intelligence platform. HR managers and coaches upload Kahler Communications PCM assessments, visualize personality profiles, analyze team dynamics, and get AI-powered coaching recommendations.

PropertyValue
Monorepoclient/ (Vite + React) + server/ (Node + Express)
Primary LanguageTypeScript — strict mode, v6.0
DatabasePostgreSQL via Prisma ORM
AuthJWT (7-day tokens, role-based)
AIMulti-provider: Claude, OpenAI, Gemini, Ollama
i18nEnglish + Russian
Dev Commandnpm run dev — starts both on ports 5173 + 3001
Default Loginadmin@pcmcoach.com / admin123

3Frontend Stack

3.1 React + Vite

Entry point: client/src/main.tsx

import { createRoot } from 'react-dom/client'
import './index.css'
import App from './App.tsx'

createRoot(document.getElementById('root')!).render(<App />)

Vite config — client/vite.config.ts:

export default defineConfig({
  plugins: [react(), tailwindcss()],
  resolve: { alias: { '@': path.resolve(__dirname, './src') } },
  server: { proxy: { '/api': 'http://localhost:3001' } },
})
Key points Path alias @/ → ./src/ — use everywhere instead of relative imports.
API calls are proxied to port 3001 in dev — no CORS config needed locally.

3.2 TypeScript (Client)

Strict mode with notable options in tsconfig.app.json:

{
  "target": "es2023",
  "module": "esnext",
  "moduleResolution": "bundler",
  "jsx": "react-jsx",
  "noUnusedLocals": true,
  "noUnusedParameters": true,
  "erasableSyntaxOnly": true,
  "ignoreDeprecations": "6.0"
}
  • noUnusedLocals/Parameters — clean up all unused variables or TS errors
  • erasableSyntaxOnly — only TypeScript syntax that erases cleanly is allowed
  • ignoreDeprecations: "6.0" — suppresses TS6 deprecation noise (also in server tsconfig)

3.3 Tailwind CSS v4

stroke="#6366f1" not stroke="hsl(var(--primary))"

⚠ React Flow v12 NodeProps data is typed as Record<string, unknown>. Always cast explicitly: String(data.label), Number(data.value) — never direct cast or type assertion.

framer-motion 12 — do not use variants stagger containers. Use direct initial/animate with transition={{ delay }}:

// ✓ Correct v12 animation pattern
<motion.div
  initial={{ opacity: 0, y: 20 }}
  animate={{ opacity: 1, y: 0 }}
  transition={{ delay: index * 0.05 }}
/>

7Auth & Security

7.1 JWT Payload

interface JWTPayload {
  userId:       string
  email:        string
  roles:        string[]
  orgId:        string | null
  tokenVersion: number  // increment to invalidate all sessions
}
// 7-day expiry, HS256, stored in localStorage as 'pcm_token'

7.2 Middleware

requireAuth               // validates Bearer token, attaches user to req
requireRole(...roles)     // 403 if user lacks ANY of the specified roles

// Passes if user has ANY listed role:
requireRole('profiles_write', 'admin')

7.3 Role Matrix

RoleAccess Level
adminFull access to everything
hrHR analytics, profiles (read-only), teams/groups
coachCoaching tools, notes, goals, profiles (read-only)
profiles_writePermission modifier — enables create/edit/delete profiles for hr/coach

7.4 Auth Cache

In-memory 60s TTL in server/src/lib/auth.ts — avoids DB round-trip on every request. invalidateAuthCache(userId) called on deactivation or role change for immediate effect. lastSeenAt updated with 5-minute throttle.

10.3 Server AI Endpoint Template

app.post('/api/my-feature', requireAuth, async (req, res) => {
  const { profileId, lang = 'en', aiConfig } = req.body
  const profile = await prisma.profile.findUnique({ where: { id: profileId } })
  if (!profile) return res.status(404).json({ error: 'Not found' })
  const result = await generateMyFeature(profile as PCMProfile, lang, aiConfig)
  await createAuditLog({ ...req.user, action: 'AI_GENERATE', entity: 'MyFeature', entityId: profileId })
  res.json(result)
})

16New Feature Implementation Blueprint

Checklist: Adding a New AI Feature

  1. Service function in server/src/services/aiRecommendations.ts:
    export async function generateMyFeature(profile, lang, config?): Promise<MyResult>
  2. AI endpoint in server/src/index.ts — follow the template in §10.3
  3. API helper in client/src/lib/api.ts
  4. Page component in client/src/pages/coaching/MyFeature.tsx — follow §10.2 template
  5. Route added to client/src/App.tsx
  6. i18n key added to client/src/i18n/en.json

Checklist: Adding a New CRUD Resource

  1. Prisma model added to schema.prisma → npm run db:push
  2. Route file created in server/src/routes/myResource.ts
  3. Route mounted in server/src/index.ts: app.use('/api/my-resource', router)
  4. API helpers added to client/src/lib/api.ts
  5. Zustand state + actions added to useStore.ts if global
  6. Page components following existing list + detail patterns
